# 🐟 Fish Ecology Assistant — 项目总览

> 把你的编码智能体变成拥有动态世界观和 DeepSeek 级工程效率的博士级研究团队。

---

## 一、这是什么

**Fish Ecology Assistant** 不是一个普通的代码仓库。它是一个**活的、有哲学的操作系统**——把 Reasonix Code 这个通用编码智能体，转变为一个专业鱼类生态学/保护遗传学研究助手。

它不是"又一个 AI 工具"。它是一套完整的、可迁移的研究工作流规范。它定义的不是代码接口，而是**人机协作的边界、知识演化的态度、以及计算资源的分配哲学**。

### 具体场景

**场景 1：文献综述**
> "研究长江禁渔对鱼类群落的影响——跑流水线。"
> → 5 引擎并行搜索中英文文献 → 分析共识演变 → 标记涌现信号 → 用 calibrated language 写出综述 → 4 维评分审查 → 输出。

**场景 2：统计咨询**
> "帮我做混合效应模型，验证手册 2.2 章。"
> → 自动查 CRAN 最新包版本 → 比对手册代码 → 概率陈旧评分 → 输出 R 代码 + 诊断 + 标注"截至 YYYY-MM 的推荐做法"。

**场景 3：跨知识库检索**
> "在 ima 知识库里搜一下同位素生态位。"
> → 自动发现 13 个知识库 → 按信息增益排序关键词 → 跨库并行搜索 → 去重 → 合成结果。

**场景 4：论文深度分析**
> "分析这篇论文。"
> → 时间轴分析（发表时→至今→当前）→ 质量评分（数据+代码+样本量+期刊）→ 涌现信号检测 → 可复现性评估。

## 三、优势

### 对比其他方案

| 维度 | 通用 ChatGPT | 通用编码智能体 | **Fish Ecology Assistant** |
|:-----|:------------|:--------------|:--------------------------|
| **领域深度** | 泛化回答 | 代码生成 | 鱼类生态学专用，含涌现检测 |
| **知识时效** | 训练数据有截止 | 不主动检查 | 自动 CRAN 版本验证 + 概率陈旧评分 |
| **搜索广度** | 单引擎 | 1-2 引擎 | 5 引擎 + 13 个 ima 知识库 |
| **输出语言** | 未校准 | 可能过度自信 | calibrated language，置信度标签 |
| **工程效率** | 不计成本 | 全量计算 | 熵预算 + 稀疏激活 + 差分更新 |
| **人机边界** | 模糊 | 不明确 | 明确定义：执行归 AI，决策归人 |
| **哲学深度** | 无 | 无 | 万物皆变：世界动态，知识暂时，涌现常态 |

### 核心差异化

**1. 动态世界观（Panta Rhei）**
不是把知识当真理，而是放在时间轴上动态看待。每个输出带时间戳，每个方法标版本号，每次分析留"下次复查"的余地。

**2. DeepSeek 级工程效率**
能量有限，算法胜暴力。熵预算分配计算资源，MoE 稀疏路由只激活需要的模块，差分验证只检查有变更的包。不求全，求准。

**3. 置信度标记与溯源**
每个声明标注来源和置信度（✅ 已验证 / ⚠️ 推断 / ❓ 不确定 / 🚫 无来源）。断链则不生成。这是 DeepSeek 式的反幻觉机制——用结构化不确定性替代蛮力检查。

**4. 人机权责边界**
执行归 AI，决策归人。AI 做搜索、分析、生成、标记涌现；人做判断真伪、选择方法、决定方向、承担发表责任。学术责任与研究品味，永远无法被算法外包。

## 四、启示

### 给研究者的启示

**知识是暂时的。** 今天写进论文的结论，三年后可能被修正。这不是失败——这是科学的常态。最好的研究工具不是那些"给出正确答案"的，而是那些**知道自己不知道什么**的。

**涌现是信号，不是噪声。** 当多个独立来源指向同一个非预期模式时，不要 dismiss 它。那可能是一个新现象的早期预警。

**工具可以外包，但品味不能。** AI 可以替你搜文献、跑代码、写草稿。但它不能替你判断什么问题是重要的，什么结论是可信的，什么创新是深刻的。这些能力需要你在真实的科研实践中，一次次地做决定，才能磨炼出来。

> 赫拉克利特说：人不能两次踏进同一条河流。
>
> 我们说：知识会老去，但人类对世界的追问永不落幕。
> 昨日之真理为今日之基石，今日之未知为明日之征途。
> 我们的目光，从不囿于已知的疆界；
> 我们的脚步，终将踏上那片星光璀璨的浩瀚征途。
