# ima 智能知识库搜索（优化版）

当用户问一个需要查资料的问题时，按此流程操作。

---

## 0. 先发现 → 后搜索（替代硬编码）

**不要依赖记忆中的 kb_id。** 先调 `ima_discover_knowledge_bases` 获取最新列表。

```yaml
流程:
  1. ima_discover_knowledge_bases() → 得到全部知识库（含最新kb_id）
  2. 从结果中提取目标领域对应的 kb_id 列表
  3. 调 ima_search_by_domain({ kb_ids: [...], query: "关键词" }) → 一次调出全部结果
```

> 💡 **为什么这样更好？** 订阅新知识库或ID变化 → 自动发现，不依赖硬编码，不过期。

---

## 1. 理解问题 → 判断领域

| 领域 | 关键词信号 |
|:-----|:-----------|
| 🐟 **鱼类学/生态学** | 鱼、鱼类、种群、群落、形态、解剖、分类、栖息地、洄游、摄食、繁殖、生长 |
| 🔬 **遗传学/形态学** | DNA、基因、SNP、遗传、形态测量、地标点、PCA、CVA、群体遗传 |
| 📊 **统计学/R语言** | 统计、模型、回归、ANOVA、t检验、p值、R代码、glmmTMB、vegan、生态数据 |
| 🤖 **机器学习/AI** | 机器学习、深度学习、随机森林、MaxEnt、SDM、神经网络、预测模型 |
| 💻 **大模型/编程** | 大模型、LLM、Transformer、GPT、Python、R、深度学习 |
| 📚 **通用/其他** | 不属于以上 → 搜全部知识库 |

## 2. 路由 → 提取 kb_id

从步骤0的发现结果中，按领域名提取 kb_id：

```
discover_result.classified["🐟鱼类学"] → [{ name, id, ... }]  ← 提取 id 列表
discover_result.classified["📊统计"]  → [{ name, id, ... }]
...
```

**如果发现结果里没有相关领域** → 用 `kb_names` 参数模糊匹配（如 `"鱼"` 自动匹配所有名称含"鱼"的库）。

## 3. 生成精准关键词

| ❌ 差 | ✅ 好 |
|:------|:------|
| "怎么做统计" | "R语言 t检验 代码" |
| "鱼" | "长江鱼类 区系 多样性" |
| "机器学习" | "随机森林 mlr3 调参" |

**同义词扩充**："混合效应模型" → `混合效应 glmmTMB mixed model`

## 4. 执行搜索（一次调用，多个库）

```yaml
ima_search_by_domain({
  kb_ids: ["id1", "id2", "id3"],  # 从步骤0提取
  query: "R语言 混合效应模型 glmmTMB",
  max_per_kb: 8                    # 每个库最多返回8条
})
```

### 无结果时
- 换同义词再搜一次（自动，不需要用户说）
- 仍无结果 → 报告"该库未找到相关内容，建议：换关键词 / 其他知识库"

## 6. 统计问题 → 自动交叉验证手册

如果用户问题是**统计方法/R代码**相关：

1. 先查 `.reasonix/handbooks/stats-methods.md` 对应章节的验证状态
2. 如果手册标记为 **"未验证"** 或验证日期较久：
   - 将 ima 搜到的代码与手册代码比对
   - 一致 → 更新验证记录为 `✅ ima知识库验证`
   - 不一致 → 标注差异，建议用 `verify-stats-handbook` 深入验证
3. 如果 ima 搜到的代码比手册的更优 → 建议用户更新手册

## 7. 学习问题 → 查阅学习手册

如果问题是**编程/AI/统计学习**相关：

1. 先查 `.reasonix/handbooks/learning-guide.md` 有没有对应主题
2. 有 → 按手册推荐的搜索词搜 ima 知识库
3. 无 → 按通用流程搜，搜到后建议补充到学习手册

## 注意事项

- **发现一次可用多次**：同一轮对话中，`ima_discover_knowledge_bases` 只需调一次，后续搜索复用结果
- **订阅库只读**：别人创建的知识库只能搜索，不能写入
- **大规模库精准搜**："统计学"库 31830 篇，关键词要具体
