## 1. 认识论循环 (Practice-Knowledge Cycle)

> 《实践论》(1937)：实践、认识、再实践、再认识，这种形式，循环往复以至无穷。

### 原理

认识的过程是从**感性认识**（数据/现象）到**理性认识**（模型/理论），再到**实践检验**（验证/预测），形成闭环。

```
感性认识 ──→ 理性认识 ──→ 实践检验 ──→ 新的感性认识 ──→ ...
 (数据)      (模型)       (验证)        (新数据)
```

### 工程映射

| 认识论环节 | 项目映射 | 实现 |
|-----------|---------|------|
| 感性认识 | 文献搜索 + 数据采集 | Executor 阶段 (5 引擎并行) |
| 理性认识 | 分析 + 模式识别 | Analyst 阶段 (涌现检测) |
| 实践检验 | 场验证 + 可复现性 | Writer→Reviewer 闭环 + 审核关卡 |
| 再认识 | 修订 + 更新 | 最多 3 轮修订，版本时间轴 |

### 运行规则

1. **任何结论必须标注验证状态**：
   - ✅ 已验证 (Verified)：经独立数据或场验证确认
   - ⚠️ 待验证 (Pending)：逻辑推断但未经验证
   - ❓ 假设 (Hypothesis)：有理据但缺乏直接证据
   - 🚫 不可验证 (Unverifiable)：禁止写入正式输出

2. **"没有调查就没有发言权"**：任何声明必须 ≥1 来源。Analyst 阶段的涌现信号需 ≥3 独立源。

3. **循环不是一次性**：复杂问题需多轮"实践→认识"循环，每轮加深理解。

## 2. 矛盾分析法 (Contradiction Analysis)

> 《矛盾论》(1937)：在复杂的事物的发展过程中，有许多的矛盾存在，其中必有一种是主要的矛盾，由于它的存在和发展规定或影响着其他矛盾的存在和发展。

### 原理

任何系统（生态系统、研究问题、技术架构）中存在多种矛盾。**主要矛盾**决定系统的性质和方向。抓住主要矛盾，一切问题迎刃而解。

### 工程映射

| 矛盾论概念 | 项目映射 | 说明 |
|-----------|---------|------|
| 主要矛盾 | 核心研究问题 | 当前研究阶段的根本问题，决定方向 |
| 次要矛盾 | 子问题/边缘问题 | 可从属于主要矛盾，或标记延后 |
| 矛盾的主要方面 | 主导因素 | 在核心问题中，哪个变量/因素起主导作用 |
| 矛盾的转化 | 研究阶段切换 | 解决主要矛盾后，次要矛盾上升为主要矛盾 |
| 对抗性矛盾 | 必须解决的冲突 | 数据矛盾、方法冲突——强制解决 |
| 非对抗性矛盾 | 可调和的分歧 | 不同研究的视角差异——标注即可 |

### 运行规则

1. **Planner 阶段必须输出矛盾分析**：
   ```
   研究问题：禁渔对长江鱼类群落的影响
   主要矛盾：禁渔政策效果 vs 其他环境因子（污染、水利工程）的混淆效应
   次要矛盾：不同江段效果差异、短期 vs 长期效应
   矛盾的主要方面：如何分离禁渔效应与其他因子的贡献
   ```

2. **资源分配按矛盾等级**：
   - 主要矛盾方向：获得 2-3x 计算资源（熵预算优先）
   - 次要矛盾方向：标准分配
   - 已标记延后的问题：跳过，仅记录

3. **矛盾转化检测**：当涌现信号积累到 high confidence（≥5 独立源）且指向新的核心问题时，自动触发重新分析——原主要矛盾已解决，新矛盾上升。

## 3. 阶段论 (Phased Strategy)

> 《论持久战》(1938)：战争可以分为三个阶段：战略防御、战略相持、战略反攻。

### 原理

面对压倒性挑战（无论是战争还是复杂研究），不可毕其功于一役。必须将过程分解为**有明确边界的战略阶段**，每个阶段有特定的目标、资源和战术。

```
战略防御 ──→ 战略相持 ──→ 战略反攻
(保存自己)    (深度耕耘)    (夺取胜利)
```

### 工程映射

| 持久战阶段 | 研究流水线映射 | 战略目标 |
|-----------|--------------|---------|
| 战略防御 | 文献调研 | 摸清现状，不急于下结论 |
| 战略相持 | 数据分析 | 积累证据，反复验证 |
| 战略反攻 | 报告撰写 + 发表 | 产出结论，影响领域 |

### 运行规则：三层目标体系

每个研究阶段必须定义三层目标：

```
战略目标 (Why)：为什么要做这个阶段？解决什么根本问题？
战役目标 (What)：这个阶段要产出什么？
战术动作 (How)：具体执行哪些操作？
```

**示例：数据分析阶段**
```
战略目标：验证禁渔是否显著改变了鱼类群落结构
战役目标：完成 β 多样性分析 + GLMM 建模
战术动作：
  1. 数据清洗 + 正态性检验
  2. 计算 Jaccard/Bray-Curtis 距离
  3. PERMANOVA 检验（禁渔前 vs 后）
  4. GLMM（固定效应：禁渔；随机效应：江段/年份）
```

### 阶段切换条件

- 不能跳跃：前一阶段未完成，不得进入下一阶段
- 可以退却：遇到死胡同时，主动退回上一阶段重新规划（"战略退却"）
- 人工把关：Field Survey 和 Conservation 阶段必须人类审批

## 4. 集中优势兵力 (Concentration of Force)

> "集中优势兵力，各个歼灭敌人"——十大军事原则

### 原理

资源总是有限的。在全局上处于劣势时，通过**在关键局部形成绝对优势**来逐一解决问题。反对"两个拳头打人"的平均主义。

### 工程映射

此原则直接增强现有的"稀疏激活"和"熵预算"机制：

| 军事原则 | 工程映射 | 实现 |
|---------|---------|------|
| 集中优势兵力 | 主要矛盾方向 2-3x token 预算 | `contradiction_analysis.contradiction_budget_multiplier` |
| 各个歼灭 | 一次解决一个问题 | Pipeline 串行，不并行处理多个主矛盾 |
| 不打无准备之仗 | 充分调查后才分析 | Executor 未返回时不激活 Analyst |
| 不打无把握之仗 | 置信度不足时不发布 | Reviewer 评分 < 阈值时强制修订 |
| 你打你的，我打我的 | 不盲从主流范式 | 独立研究路径，不因"别人都这样做"而跟风 |

### 运行规则

1. **熵预算分配优先序**：
   ```
   主要矛盾方向：60% 计算资源
   次要矛盾方向：30% 计算资源
   边缘问题：     10% 计算资源
   ```

2. **"你打你的，我打我的"规则**：
   当主流文献都使用某种方法时，系统不应自动跟随。应评估：
   - 该方法是否适用于当前研究的具体条件？
   - 是否有更适合的替代方法？
   - 输出应标注："主流方法为 X，但基于本研究的特点（...），选择方法 Y"

## 5. 主动权 / 能动性 (Initiative & Agency)

> 《论持久战》：主动性，说的是军队行动的自由权……失去了这种自由，军队就接近于被打败或被消灭。

### 原理

在任何竞争或探索过程中，**主动权**是关键。被动应对意味着被对方（或环境）牵着走。必须主动塑造条件、主动提出问题、主动选择战场。

### 工程映射

| 能动性表现 | 项目映射 |
|-----------|---------|
| 主动提出问题 | Frontier Tracker 主动建议研究方向 |
| 主动选择战场 | 研究问题拆解时自主选择切入角度 |
| 主动塑造条件 | 方法选择基于问题特点，不被主流束缚 |
| 不要被动应付 | 不只做"用户问了什么就搜什么"的反应式工具 |

### 运行规则

1. **前沿追踪主动建议**：`frontier-tracker` 每轮定期输出"建议研究方向"（3 个），标注矛盾等级
2. **反向提问**：当用户问题过于宽泛时，系统应主动追问以明确主要矛盾
3. **"战略上藐视，战术上重视"**：
   - 战略层：相信问题可解，大胆设定目标
   - 战术层：每个步骤严格验证，审慎执行

## 6. 矛盾分类处理 (Differentiated Contradiction Handling)

> 《关于正确处理人民内部矛盾的问题》(1957)：敌我之间和人民内部这两类矛盾的性质不同，解决的方法也不同。

### 原理

不是所有问题都需要同样的处理方式。对抗性矛盾（根本冲突）需要彻底解决；非对抗性矛盾（内部差异）需要调和与协调。

### 工程映射

| 矛盾类型 | 研究映射 | 处理策略 |
|---------|---------|---------|
| 对抗性矛盾 | 数据逻辑冲突、方法失效 | 彻底排查，强制解决 |
| 非对抗性矛盾 | 不同研究视角差异 | 标注并存，等待更多证据 |
| 结构性矛盾 | 资源/时间约束 | 优化配置，分阶段解决 |
| 阶段性矛盾 | 当前阶段特有 | 标记，随阶段推进自然消解 |

### 运行规则

1. **矛盾分类标签**（在 Analyst 阶段输出）：
   - 🔴 对抗性：必须解决，阻塞后续阶段
   - 🟡 非对抗性：标注但不阻塞
   - 🔵 结构性：记录，分配资源时考虑
   - ⚪ 阶段性：标记，阶段切换时复核

2. **解决策略分流**：
   - 对抗性矛盾 → 触发强制分析（调用 reasoning 模型，增加 token 预算）
   - 非对抗性矛盾 → 标注 + 纳入不确定性输出
   - 结构性矛盾 → 记录，输出时注明约束条件

## 7. 十大关系式系统平衡 (Multi-Factor Balancing)

> 《论十大关系》(1956)：提出要正确处理重工业和轻工业/农业、沿海和内地、经济建设和国防建设等十大关系。

### 原理

复杂系统不是一个维度的问题，而是多个相互关联、相互制约的因素构成的网络。不能只抓一点不及其余——需要**统筹兼顾**。

### 工程映射——研究中的十大平衡

| # | 关系对 | 研究映射 |
|---|--------|---------|
| ① | 广度 vs 深度 | 多引擎搜索 vs 单篇深入分析 |
| ② | 速度 vs 质量 | 快速出报告 vs 严格验证 |
| ③ | 英文 vs 中文 | 国际文献 vs 本地知识 |
| ④ | 定量 vs 定性 | 统计显著 vs 生态意义 |
| ⑤ | 新方法 vs 经典方法 | 前沿技术 vs 经过验证的标准 |
| ⑥ | 计算 vs 人工 | AI 自动 vs 人类判断 |
| ⑦ | 发表压力 vs 科学严谨 | 快速产出 vs 充分验证 |
| ⑧ | 局部 vs 全局 | 单一江段/种群 vs 全流域 |
| ⑨ | 当下 vs 历史 | 最新文献 vs 经典基础 |
| ⑩ | 已知 vs 未知 | 确定性结论 vs 不确定性标注 |

### 运行规则

1. **每阶段输出应隐含平衡检查**：
   - 是否过度偏向一方？
   - 是否有被忽略的对立面？

2. **多目标优化而非单目标**：当矛盾无法同时最优时，遵循：
   ```
   科学严谨 > 速度
   生态意义 > 统计显著
   本地知识 ≥ 国际文献（鱼类生态学领域）
   ```

### 控制论闭环 (Cybernetic Feedback)

Agent 的运行时数据流构成一个**控制论反馈系统**，与《实践论》的认识论循环完全对应：

| 控制论环节 | 实践论环节 | Agent 层 | 工程实现 |
|-----------|-----------|---------|---------|
| 感知 (Sensor) | 感性认识 | 交互与感知层 | 接收用户输入/环境数据 |
| 决策 (Controller) | 理性认识 | 认知与决策层 | LLM 推理 + 任务分解 |
| 执行 (Actuator) | 实践 | 工具与执行层 | MCP 调用 / 沙盒执行 |
| 反馈 (Feedback) | 再认识 | 执行→认知回路 | 报错/成功信息流回认知层 |
| 调节 (Adaptation) | 再实践 | 认知→映射回路 | 修正后的策略重新执行 |

### Panta Rhei 与闭环反馈

| Panta Rhei 原则 | 控制论对应 | 5 层 Agent 实现 |
|----------------|-----------|----------------|
| 🌍 世界是动态的 | 环境状态 $O_t$ 随时间变化 | 感知层持续接收新观察 |
| 📖 知识是暂态的 | Belief 更新函数 $\Phi$ | 记忆系统层更新信念状态 |
| 🔬 涌现是常态 | Emergence 检测 | 认知层涌现检测 → 矛盾转化 |
| 效率即智能 | 最小熵增路径 | 稀疏激活 + 冗余检测 |

```
Systems Thinking 七原则         标准 5 层 Agent 架构       运行时数据流
─────────────────────    ─────────────────────────    ─────────────────
① 认识论循环 (实践→认识)  →  ①交互感知 → ②认知 → ⑤执行 → ②再认知  闭环反馈
```
